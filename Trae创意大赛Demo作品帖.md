# 【正文】

## 1. Demo 简介

**是什么：** 一款纯前端网页游戏（网站），打开浏览器即可玩，无需下载安装。用电脑键盘或手机触屏弹奏钢琴，支持跟弹打分和自由弹奏双模式，内置 AI 音频转谱功能——上传本地音频文件即可自动生成乐谱和按键提示，配 Three.js 3D 钢琴视觉、粒子特效、多风格背景与摄像头画中画，风格用户可自由切换。零后端、零安装，单 HTML 压缩包提交即体验。

**面向谁：** 男女老少皆宜。核心用户三类——① 喜欢音乐但没学过钢琴的普通人（学生、上班族，想放松解压）；② 有一定钢琴基础、想用碎片时间练琴的爱好者；③ 对 AI 技术好奇、想体验"上传首歌就能自动生成乐谱"的科技尝鲜者。

**主要功能：**

**功能一：键盘弹奏 + 跟弹打分**

用电脑键盘（A/S/D/F/G/H/J 弹白键，W/E/T/Y/U 弹黑键，Z/X 切换八度）或手机触屏弹奏钢琴。跟弹模式下，音符从屏幕上方下落，到达判定线时按下对应键即可得分，判定分为 Perfect / Great / Good / Miss 四档，配有连击系统和结算评分。Web Audio API 保证按下到出声延迟 < 50ms，支持钢琴、音乐盒、Pad 等多种音色。

**功能二：AI 音频转谱**

上传本地 MP4 视频或音频文件（WAV/MP3/OGG/FLAC），浏览器端 Spotify Basic Pitch 神经网络自动识别音高与节奏，生成 MIDI 音符数据，再转换为五线谱（OpenSheetMusicDisplay 渲染）和下落式按键提示。用户无需任何乐理知识，也不需要找 MIDI 文件——上传一首喜欢的歌，就能立刻跟着弹。全程音频不出本机，隐私零泄露。

**功能三：沉浸式 3D 视觉**

Three.js 渲染 3D 钢琴键盘与粒子系统，按键时触发粒子爆发特效。背景风格可在"星空""樱花""赛博朋克"三种主题间切换，还可开启摄像头画中画功能，看到自己弹琴的画面。视觉风格与音乐氛围融为一体，把"敲键盘"变成"在演奏厅弹琴"的沉浸体验。

## 2. Demo 创作思路

**灵感来源：**

很多人在听到喜欢的歌时，会下意识在桌面上用手指"弹空气钢琴"，或者随手在键盘上敲击试图还原旋律。这个本能动作说明，"想弹琴"的冲动人皆有之，只是被乐器门槛挡住了。如果能把这个动作变成真正能出声、能打分、能学习的游戏，再配上 AI 自动把任意音频转成可弹的乐谱，就能让任何人零门槛享受弹琴的乐趣。

**想解决的问题：**

- **学琴门槛高：** 传统钢琴动辄上万元，线下课程按小时收费，识谱需要长期练习，大量想"随便弹弹"的人被挡在门外
- **现有产品不够"傻瓜"：** Synthesia 需要用户自己找 MIDI 文件导入，普通人不知道 MIDI 是什么；Virtual Piano 只能自由弹奏没有引导和打分，弹得对不对全靠自己听
- **没有"听歌即弹"的能力：** 用户听到一首歌想弹，现有工具无法自动把音频变成可弹的乐谱，手动扒谱需要专业听力
- **缺乏沉浸感：** 多数网页钢琴就是一张静态琴键图，没有视觉氛围，弹两下就无聊了

**为什么做这个方向：**

三个判断——① 音乐娱乐是刚需，"放松解压"覆盖男女老少，受众面极广；② Spotify 开源了 Basic Pitch 纯前端 AI 转谱库，让"浏览器端 AI 听歌写谱"在技术上首次成为可能，这是产品差异化的核心技术壁垒；③ 纯前端架构天然契合大赛"HTML 压缩包提交"的形式，零后端零部署，评委开包即体验，音频全程本地处理也规避了隐私问题。三者叠加，这个方向既有技术亮点，又有落地可行性，还能真正服务普通用户。

### 4.2 关键技术决策

| 决策点 | 选择 | 理由 |
|---|---|---|
| AI 运行位置 | 纯前端 Spotify Basic Pitch TS 库 | 满足 HTML 压缩包提交 + 隐私 + 零后端 |
| 音源方案 | Web Audio API 合成 + SoundFont 采样 | 兼顾音质与体积，按下到出声 < 50ms |
| MP4 音频提取 | 引导直传音频优先 + ffmpeg.wasm 单线程懒加载兜底 | 规避 SharedArrayBuffer/COOP-COEP 头要求 |
| 乐谱渲染 | OpenSheetMusicDisplay（基于 VexFlow） | 官方支持 MusicXML 标准 |
| 打分判定窗口 | Perfect ±50ms / Great ±100ms / Good ±150ms | 平衡难度与娱乐性 |
| 键盘映射 | 双行映射（白键 ASDFGHJ + 黑键 WETYU）+ Z/X 切八度 | 88 键折叠到可用物理按键 |
| 数据存储 | IndexedDB + localStorage | 纯前端持久化，断网可用 |
| 摄像头 | 默认关闭 + 首次隐私弹窗 + 拒绝则隐藏 | 降低敏感度，优雅降级 |

### 4.3 开发心得

- **AI 转谱的"娱乐向"定位很关键：** Basic Pitch 对单乐器清晰录音效果最佳，复杂混音准确率会下降。PRD 中明确将 AI 定位为"娱乐向自动转谱"，UI 标注"识别结果仅供参考"，并设计了分级降级策略（模型加载失败→引导用内置曲库；推理超时→可继续等待/取消），避免用户预期落差。
- **纯前端的隐私优势是卖点：** 所有音频处理在浏览器本地完成，不上传任何服务器，这在隐私意识越来越强的今天是实实在在的竞争优势，也简化了大赛提交（无需部署后端）。
- **TRAE 的技能系统大幅提效：** `doc-writing-guide` 提供了 PRD 写作规范与反模式清单，`html-report` 提供了自包含 HTML 生成脚手架（字体/JS 库/目录结构），让需求文档从大纲到可交互 HTML 只用了几轮对话。
